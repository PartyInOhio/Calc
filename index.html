Windows Server Licensing
Quick reference
Standard Edition
Covers 2 Windows VMs per license. Licensed per physical core — minimum 16 cores per server. Stack multiple licenses for more VMs.
Datacenter Edition
Covers unlimited Windows VMs. Same per-core licensing model. Better value when running 3+ Windows VMs on a host.
ℹ️ Linux VMs and non-Windows guests do not consume Windows Server licenses. Licensing applies per physical host.
🗄️
SQL Server Licensing
Core-based per-VM model
Standard Edition
Per-VM core licensing. Min 4 cores/VM. Capped at 4 sockets / 24 cores and 128 GB RAM per instance.
Enterprise Edition
Per-VM core licensing. Min 4 cores/VM. No limits on cores, sockets, or RAM. Required for large workloads.
ℹ️ Software Assurance (SA) required for license mobility across hosts. Consider Azure Hybrid Benefit if moving to cloud.

Use Cases
RAID 0 — Performance only
Zero redundancy. Any single disk failure = total data loss. Use for scratch/temp only.
RAID 1 — Boot / OS drives
Full mirror. Maximum protection, half the capacity. Ideal for small critical volumes.
RAID 5 — NAS & file servers
Best balance of capacity, throughput, and single-disk redundancy.
RAID 6 — Large arrays (>6 disks)
Dual parity protects against 2 simultaneous failures — essential when rebuild time is long.
RAID 10 — High-I/O databases
Best write performance with redundancy. Fixed 50% efficiency is the cost.
ℹ️ Usable values are theoretical. Real-world capacity is slightly less due to filesystem overhead. RAID is not a backup.
